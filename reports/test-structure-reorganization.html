Test Structure Reorganization

Durable frontend/domain tests have been moved out of colocated implementation folders and into the new curated src/test structure. Colocated tests remain allowed while an agent is building, but before commit they should be deleted, converted to E2E, or moved into this durable tree.

Process Update

  • Agents may write colocated tests beside changed files during implementation.
  • Before handoff or commit, run bun run test:cull.
  • Temporary or low-signal colocated tests should be deleted.
  • Durable frontend/domain coverage should move into src/test.
  • User-visible flows should move toward e2e/ smoke coverage.
  • API and script tests can remain colocated with their route/script boundary.
The cull script now treats src/test as the curated durable tree, so moved tests are reported as keep candidates with an explicit reason instead of being re-flagged by filename heuristics.
